1. user

   Таблица предназначена для хранения информации обо всех пользователях интернет-магазина, как его сотрудниках так и покупателях.
   
   Атрибутный состав:
   
   |Наименование поля|Ключ|Тип данных|Уникальность|Обязательность|Значение по умолчанию|Описание поля|
   |-----------------|----|----------|------------|--------------|---------------------|-------------|
   |id|PK|int|да|да|автоинкремент|уникальный идентификатор записи|
   |email||varchar(128)|да|да||адрес электронной почты, выполняет функцию логина - имени пользователя в системе|
   |first_name||varchar(128)|нет|да||Имя пользователя|
   |surname||varchar(128)|нет|нет||Отчество пользователя|
   |last_name||varchar(128)|нет|да||Фамилия пользователя|
   |password||varchar(128)|нет|да||Хэш пароля|
   |is_active||bool|нет|да|false|Признак активности пользователя. Пользователь становится активным после подтверждения адреса эл.почты. Может быть заблокирован сотрудником интернет-магазина|
   |phone_number||varchar(128)|да|да||Контактный телефон пользователя.|
   |personal_discount||int|нет|да|0|Персональная скидка покупателя|

2. user_address

   Таблица предназначена для хранения адресов доставки заказа покупателю. Один пользователь может иметь несколько адресов доставки. Один из них может являться дефолтным.
   
   Атрибутный состав:
   
   |Наименование поля|Ключ|Тип данных|Уникальность|Обязательность|Значение по умолчанию|Описание поля|
   |-----------------|----|----------|------------|--------------|---------------------|-------------|
   |id|PK|int|Да|Да|Автоинкремент|Первичный ключ|
   |user_id|FK|int|нет|да||ссылка на учетную запись пользователя|
